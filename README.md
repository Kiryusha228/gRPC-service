Для запуска Tarantool использовал docker-compose, Из библиотеки tarantool-java-sdk использовался TarantoolBoxClient, с помощью которого методом .call вызывались определенные в init.lua файле функции.
Для Tarantool создается отдельный пользователь, с правами на спейс и запуск определенных в нем функций.

gRPC использовался из коробки со start.spring.

Внутри сервиса gRPC планировался вызов crud слоя, однако во время разработки пришел к выводу что еще один слой абстракции для настолько маленького проекта будет антипаттерном.

Т.к. подразумевается что работа будет с огромным объемом данных метод range работает при помощи батчей, размер которых указывается в .env файле
